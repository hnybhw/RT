打开VS Code终端：Ctrl+` 

git add .
解释：. 代表“当前文件夹”，意思是把当前文件夹下的所有文件都添加到 Git 的暂存区
预期效果：没有输出是正常的

git commit -m "初始化 RT 项目"	
解释：-m 后面跟的是本次提交的说明。
预期效果：会显示一些信息，比如创建了多少个文件，插入了多少行代码

git remote add origin https://github.com/hnybhw/RT.git
解释：把你的本地仓库和 GitHub 上的远程仓库连接起来（这个你之前可能已经做过一次了，如果提示 remote origin already exists，可以先跳过，或者先运行 git remote remove origin 再重新添加）。
预期效果：没有输出是正常的

git push -u origin master
解释：把本地 master 分支的代码推送到远程仓库。
看到 master 了吗？你本地的默认分支名叫 master，不是 main！ 这是 Git 的老传统（master）和新传统（main）的区别。
预期效果：会显示上传进度，最后提示 main -> main 之类，就说明成功了

完成后，刷新你的 GitHub 仓库页面 https://github.com/hnybhw/RT，就能看到你本地的所有文件和文件夹了。

日常同步流程（三步走）

每次你在 VS Code 里修改了代码，想要同步到 GitHub，就按这个流程操作：

第一步：查看修改了哪些文件
点击 VS Code 左侧的 “源代码管理” 图标（🔀，就是那个分支形状的）
你会看到 “Changes”（更改）区域下面列出了所有你修改过的文件
M = Modified（修改过的文件）
U = Untracked（新增的文件）
D = Deleted（删除的文件）

第二步：提交修改到本地仓库
在 “源代码管理” 面板顶部的输入框里，写下你对这次修改的说明，例如：优化 C10KProcessor 的计算性能
点击输入框上方的 “Commit” 按钮（✓ 对勾图标）
如果你只想提交部分文件，可以右键点击具体文件 → “暂存更改”，然后再 Commit

第三步：推送到 GitHub
提交完成后，你会看到底部有一个 “同步更改” 按钮（两个循环箭头 🔄）
点击它，VS Code 就会把你的本地提交推送到 GitHub 上
也可以点击“...”菜单，选择 “推送”
完成！ 刷新你的 GitHub 仓库页面，就能看到最新的代码了。

如果你习惯用命令行，在 VS Code 终端里执行：
# 查看当前修改状态
git status
# 添加所有修改到暂存区
git add .
# 提交修改，附上说明
git commit -m "这里写你的修改说明"
# 推送到 GitHub
git push

什么时候该 Commit？
场景	是否 Commit	提交说明示例
完成一个小功能	✅ 是	新增超额赔款计算逻辑
修复了一个 bug	✅ 是	修复 asmp 解析时的空指针错误
优化了代码结构	✅ 是	重构 aggregator，提取公共方法
核心原则：每次 Commit 应该是一个完整的、有意义的修改单元，并且代码至少是能跑通的。


一个小技巧：Commit 前先 Pull
如果你在多台电脑上工作（比如公司和家里），推送前最好先拉取一下远程的最新代码，避免冲突：
git pull
如果提示有冲突，VS Code 会用红色标记出来，手动解决后再 Commit + Push。
